CRON: daily news brief — **these instructions override any default “newsletter” style.** Deliver to Telegram **only** the Markdown document described below. **No** greeting (“Hey…”, “Good morning”), **no** emoji in headings, **no** themed sections (Tech / Sports / World), **no** footnotes about `/tmp`, cron, or debugging.

**Source (only):** GET `http://127.0.0.1:8080/i/?a=rss&user=admin&nb=100&search=pubdate:PT48H/` and **read the response as XML** yourself (`<item>` → `title`, `link`, `pubDate`, `description`). **No** Python/shell to parse the feed for you.

**Steps:** (1) Pick **3** stories for deep dive + **7–10** for short list using memory/profile. (2) For each of the **3** deep items, **web-fetch** at least one article `link` **one URL at a time**. If a `link` is a **Google News** wrapper (`news.google.com` with `/rss/articles/`, `/articles/`, or `/read/`), apply **`resolve-google-news-url`** skill first, then run `python3 scripts/resolve_google_news_url.py '<url>'` from **`resolve-google-news-url/`** and **web-fetch the printed publisher URL**; use that URL in *Links:* too. (3) Write **only** the Markdown below. Entire brief in the user’s **native language** (labels translated).

**Required Markdown shape — use exactly two `###` sections plus `Focus today`; nothing else before or after:**


Omit the **Gaps** line if empty. Deep bullets must be **paragraphs**, not one-liners.

**Invalid:** any other layout; fewer/more than 3 deep stories; missing any of the four bullets under items 1–3; links missing.
